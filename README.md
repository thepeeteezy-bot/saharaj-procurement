# ระบบใบสั่งซื้อ–สั่งจ้าง งานพัสดุ โรงเรียนสหราษฎร์รังสฤษดิ์

ระบบนี้ใช้สถาปัตยกรรม:

- หน้าเว็บ: HTML + CSS + JavaScript
- Hosting: GitHub Pages
- Backend/API: Google Apps Script Web App
- ฐานข้อมูล: Google Sheet

## โครงสร้างหน้าเว็บ

- `index.html` — หน้าหลัก
- `purchase.html` — แบบบันทึกใบสั่งซื้อ
- `hire.html` — แบบบันทึกใบสั่งจ้าง
- `records.html` — รายการเอกสาร ประกอบด้วย  
  เลขที่, วันที่, เรื่อง, จำนวนเงิน, สถานประกอบการ, เลขผู้เสียภาษี, เลขที่โครงการ

---

## 1. สร้าง Google Sheet

1. สร้าง Google Sheet ใหม่
2. ตั้งชื่อ เช่น `ระบบใบสั่งซื้อ-สั่งจ้าง`
3. คัดลอก Spreadsheet ID จาก URL

ตัวอย่าง URL:

```text
https://docs.google.com/spreadsheets/d/XXXXXXXXXXXXXXXXXXXXXXXX/edit
```

Spreadsheet ID คือข้อความระหว่าง `/d/` และ `/edit`

ไม่จำเป็นต้องสร้างหัวตารางเอง เพราะ Apps Script จะสร้าง Sheet ชื่อ `Documents` และหัวตารางให้อัตโนมัติ

---

## 2. ติดตั้ง Google Apps Script

1. ใน Google Sheet ไปที่ **ส่วนขยาย > Apps Script**
2. เปิดไฟล์ `google-apps-script/Code.gs`
3. คัดลอกโค้ดทั้งหมดไปวางใน Apps Script
4. แก้บรรทัดนี้:

```javascript
const SPREADSHEET_ID = 'PASTE_YOUR_GOOGLE_SHEET_ID_HERE';
```

ให้เป็น Spreadsheet ID ของคุณ

5. ตั้ง Time zone ของโครงการเป็น `Asia/Bangkok`
6. กด **Deploy > New deployment**
7. เลือกชนิด **Web app**
8. Execute as: **Me**
9. Who has access: **Anyone**
10. กด Deploy และอนุญาตสิทธิ์
11. คัดลอก URL ที่ลงท้ายด้วย `/exec`

---

## 3. ตั้งค่า URL ในเว็บไซต์

เปิดไฟล์:

```text
assets/config.js
```

แก้ค่า:

```javascript
API_URL: "PASTE_YOUR_GOOGLE_APPS_SCRIPT_WEB_APP_URL_HERE"
```

เป็น URL ของ Apps Script Web App

---

## 4. อัปโหลดขึ้น GitHub Pages

1. สร้าง GitHub repository ใหม่ เช่น `saharaj-procurement`
2. อัปโหลดไฟล์และโฟลเดอร์ทั้งหมดในชุดนี้
3. ไปที่ **Settings > Pages**
4. Source เลือก **Deploy from a branch**
5. Branch เลือก `main` และโฟลเดอร์ `/root`
6. กด Save

เว็บไซต์จะมีรูปแบบประมาณ:

```text
https://USERNAME.github.io/saharaj-procurement/
```

## ข้อควรระวัง

เวอร์ชันนี้เป็นระบบภายในแบบพื้นฐาน หน้าแบบฟอร์มเปิดให้ผู้ที่มีลิงก์สามารถเพิ่มข้อมูลได้ จึงควร:

- ใช้ลิงก์ภายในหน่วยงาน
- ไม่ใส่ข้อมูลลับหรือข้อมูลส่วนบุคคลที่ไม่จำเป็น
- สำรอง Google Sheet เป็นระยะ
- หากต้องการจำกัดสิทธิ์ ควรเพิ่ม Google Sign-In หรือระบบตรวจสอบบัญชีผู้ใช้ในระยะต่อไป


## รายการล่าสุดบนหน้าแรก

หน้า `index.html` จะแสดงรายการที่บันทึกล่าสุดจาก Google Sheet ไม่เกิน 8 รายการโดยอัตโนมัติ
และมีปุ่ม `ดูรายการทั้งหมด` เพื่อเปิดหน้า `records.html`


## รายการแยกตามหน้าแบบบันทึก

- หน้า `purchase.html` แสดงเฉพาะรายการใบสั่งซื้อ
- หน้า `hire.html` แสดงเฉพาะรายการใบสั่งจ้าง
- เมื่อบันทึกข้อมูลสำเร็จ ตารางในหน้านั้นจะโหลดข้อมูลใหม่อัตโนมัติ


## ฟอนต์ของระบบ

เว็บไซต์กำหนดฟอนต์หลักเป็น `TH Niramit AS` และมีฟอนต์สำรอง `TH Niramit`, `Niramit`, `Tahoma` ตามลำดับ
หากเครื่องผู้ใช้ไม่มี TH Niramit AS เบราว์เซอร์จะเลือกใช้ฟอนต์สำรองอัตโนมัติ


## ขนาดตัวอักษร

เวอร์ชันนี้เพิ่มขนาดตัวอักษรทั้งระบบ โดยเฉพาะหัวข้อ แบบฟอร์ม ปุ่ม และตาราง เพื่อให้อ่านง่ายขึ้นบนคอมพิวเตอร์และมือถือ


## ฟอนต์ TH Sarabun New

เวอร์ชันนี้ใช้ `TH Sarabun New` เป็นฟอนต์หลัก พร้อมเพิ่มขนาดตัวอักษรสำหรับหัวข้อ แบบฟอร์ม ปุ่ม และตาราง
หากเครื่องผู้ใช้ไม่มีฟอนต์ TH Sarabun New ระบบจะใช้ `Sarabun` หรือ `Tahoma` เป็นฟอนต์สำรอง


## การปรับหน้าแรก

เวอร์ชันนี้นำส่วน `ข้อมูลที่จัดเก็บ` ออกจากหน้าแรกแล้ว โดยคงเมนูใบสั่งซื้อ ใบสั่งจ้าง และรายการล่าสุดไว้ตามเดิม


## ธีมการศึกษาดิจิทัล

เวอร์ชันนี้ปรับภาพลักษณ์เว็บไซต์ให้เป็นธีมการศึกษาดิจิทัล โดยใช้โทนสีฟ้า–น้ำเงิน–เขียว
มีพื้นหลังลายกริดดิจิทัล การ์ดและปุ่มสมัยใหม่ และจัดรูปแบบให้ทุกหน้าดูเป็นระบบเดียวกัน


## ธีมโรงเรียนดิจิทัลสีเขียว

หน้าเว็บใช้โทนสีเขียว พร้อมภาพประกอบโรงเรียน ครู และนักเรียนที่ใช้อุปกรณ์ดิจิทัล
ไฟล์ภาพอยู่ที่ `assets/images/digital-school-green.svg`


## ธีมส้ม Minimal

เวอร์ชันนี้ลบภาพ `digital-school-green.svg` ออกจากหน้าแรก และปรับเว็บไซต์เป็นโทนสีส้มแบบมินิมอล
โดยยังคงฟอนต์ TH Sarabun New และขนาดตัวอักษรใหญ่สำหรับการอ่านที่ชัดเจน


## ตราโรงเรียน

เพิ่มตราโรงเรียนในส่วนหัวของหน้า `index.html`, `purchase.html`, `hire.html` และ `records.html`
ไฟล์ตราอยู่ที่ `assets/images/school-logo.png`


## ช่องฝ่ายงานที่รับผิดชอบ

เพิ่มช่อง `ฝ่ายงานที่รับผิดชอบ` ในหน้าใบสั่งซื้อและใบสั่งจ้าง และแสดงในหน้าแรก หน้ารายการเอกสาร และตารางประวัติแต่ละประเภท

ก่อน Deploy `Code.gs` เวอร์ชันใหม่ ให้ใส่ Spreadsheet ID เดิมของระบบลงในตัวแปร `SPREADSHEET_ID` จากนั้น Deploy เป็น New version ระบบจะเพิ่มคอลัมน์ `department` และจัดลำดับคอลัมน์โดยไม่ลบข้อมูลเดิม


## ธีมสีม่วง–แดง

เวอร์ชันนี้ปรับเว็บไซต์เป็นธีมสีม่วง–แดงตามสีประจำโรงเรียน โดยคงตราโรงเรียน ฟอนต์ TH Sarabun New ขนาดใหญ่ และฟังก์ชันทั้งหมดไว้ตามเดิม


## การกรอกฝ่ายงานที่รับผิดชอบ

ช่อง `ฝ่ายงานที่รับผิดชอบ` เป็นช่องข้อความอิสระ ผู้ใช้สามารถกรอกข้อความใดก็ได้
แต่ระบบจะไม่อนุญาตให้บันทึกเมื่อเว้นว่างหรือกรอกเฉพาะช่องว่าง
